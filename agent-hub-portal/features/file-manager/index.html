File Manager

Unified file management with scope-based storage and TTL support

Version: 1.5.0 Status: Active Run Tests →

Overview

The File Manager provides a unified system for handling files across different storage scopes with automatic TTL management. It supports temporary files for code execution, session files, agent-specific files, team resources, and permanent company documents.

Storage Scopes

Scope TTL Storage Use Case
temporary 10 min Memory Code execution outputs
session 24 hours Disk Session working files
agent 7 days Disk Agent test files
team 30 days GCP Team resources
company Permanent GCP Long-term documents

API Endpoints

POST /files/upload

Upload file with specified scope and metadata

GET /files/:id/download

Download file (no authentication required)

GET /files/list

List files by scope with filtering

GET /files/agent/:agentId

Get agent-specific files

DELETE /files/:id

Delete file with permission checks

Key Features

  • ✓ Automatic TTL management with cleanup
  • ✓ Multiple storage backends (Memory, Disk, GCP)
  • ✓ Scope-based access control
  • ✓ Integration with code execution service
  • ✓ Support for file URLs and base64 content
  • ✓ Metadata and tagging support
