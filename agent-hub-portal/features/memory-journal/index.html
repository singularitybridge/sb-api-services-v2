Memory/Journal

Persistent memory storage and journal entries for AI assistants

Version: 2.0.0 Status: Active Run Tests →

Overview

The Memory/Journal feature provides a sophisticated system for AI assistants to store and retrieve contextual information across sessions. It enables persistent memory storage, journal entries for tracking interactions, and intelligent retrieval mechanisms.

API Endpoints

POST /memory/store

Store a new memory entry with tags and metadata

GET /memory/retrieve

Retrieve memories by context, tags, or date range

POST /journal/entry

Create a new journal entry for session tracking

GET /journal/history

Retrieve journal history with filtering options

DELETE /memory/:id

Delete specific memory entries

Key Features

  • ✓ Vector-based semantic search for memory retrieval
  • ✓ Tag-based categorization system
  • ✓ Time-based memory decay and relevance scoring
  • ✓ Journal entries with rich metadata
  • ✓ Cross-session context preservation
  • ✓ Privacy-focused with user-scoped data
