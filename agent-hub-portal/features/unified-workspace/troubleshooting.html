Troubleshooting Guide

Common issues and solutions for Unified Workspace

Run Tests

Issues

Quick Fix

For most issues, try:

export USE_MEMORY_STORAGE=true

This bypasses MongoDB completely.

Common Issues

Test page freezes when clicking "Run Test"

Symptoms: Browser becomes unresponsive, no response from API

Solution:

  1. Set environment variable: USE_MEMORY_STORAGE=true
  2. Restart the server
  3. Clear browser cache and reload the test page

Root Cause: MongoDB connection timeout. The fix adds 5-second timeouts to all database operations.

401 Unauthorized Error

Symptoms: API returns 401 even with valid API key

Solution:

Ensure your API key includes the Bearer prefix:

Authorization: Bearer sk_live_your_api_key_here

Not just: sk_live_your_api_key_here

Files not persisting after server restart

Symptoms: Files disappear when using in-memory storage

Solution Options:

  1. Configure MongoDB: Set MONGODB_URI environment variable
  2. Use local file storage: Files persist in /tmp/workspace
  3. Configure S3: Set AWS credentials for cloud storage

Authentication Issues

API Key Format

❌ Incorrect:

Authorization: sk_live_e69dfeb21e6eb4c5b2d17e92b0ff0a4d

✅ Correct:

Authorization: Bearer sk_live_e69dfeb21e6eb4c5b2d17e92b0ff0a4d

Testing Authentication

curl http://localhost:3000/health \
  -H "Authorization: Bearer sk_live_your_api_key"

The test page automatically adds the Bearer prefix if you enter just the API key.

Storage Issues

Storage Modes

In-Memory Storage

Best for development and testing

export USE_MEMORY_STORAGE=true
  • ✅ No external dependencies
  • ✅ Fast performance
  • ❌ Data lost on restart
  • ❌ Limited by RAM

MongoDB Storage

Best for production with persistence

export MONGODB_URI=mongodb://localhost:27017/workspace
  • ✅ Data persists
  • ✅ Scalable
  • ✅ 5-second timeout protection
  • ❌ Requires MongoDB setup

Local File Storage

For large files and binary content

export LOCAL_STORAGE_PATH=/var/workspace
  • ✅ Handles large files
  • ✅ No database needed
  • ✅ Default: /tmp/workspace
  • ❌ Single server only

Tip: The system automatically routes content based on size:

  • • < 10KB → Memory cache
  • • 10KB - 1MB → MongoDB
  • • > 1MB → File storage

Performance Issues

Slow Response Times

Check MongoDB Connection

If responses take exactly 5 seconds, MongoDB is timing out.

# Switch to in-memory storage export USE_MEMORY_STORAGE=true

Monitor Cache Usage

Check workspace info for cache statistics:

curl http://localhost:3000/api/workspace/info \ -H "Authorization: Bearer sk_live_..."

Optimize Cache Size

For heavy usage, increase cache size:

  • • Default: 1,000 items (MongoDB mode)
  • • Default: 10,000 items (in-memory mode)
  • • Adjust in unified-workspace.service.ts

Memory Usage

Monitor memory usage with workspace info endpoint:

{
  "totalItems": 23,
  "cacheSize": 23,
  "maxCacheSize": 10000,
  "storageMode": "in-memory"
}

Warning: Cache automatically prunes when reaching 80% capacity using LRU eviction.

Testing Issues

Running Tests

Browser Testing

  1. Navigate to: http://localhost:8083/agent-hub-portal/features/unified-workspace/tests/
  2. Click "Configure" to set API URL and key
  3. Enter API key (without Bearer prefix)
  4. Click individual test buttons

Command Line Testing

# With in-memory storage
export USE_MEMORY_STORAGE=true
node test-unified-workspace.js

# With custom API key
export API_KEY=sk_live_your_key
node test-unified-workspace.js

Common Test Failures

  • Move/Copy operations fail:

    Use fromPath/toPath parameters, not from/to

  • List returns empty:

    In-memory mode only lists cached items

  • Clear operations fail:

    Expected - cleanup operations may fail in test mode

Test Success Rate
